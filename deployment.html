Deployment Guide

Step-by-step instructions
  1. Fork the Repository: First, fork the official CIPTER-MD repository to your own GitHub account.
  2. Get Session ID: Use our 'Connect Bot' page to scan the QR code or use a Pairing Code to generate your unique SESSION_ID.
  3. Connect Platform: Click one of the Auto Deploy buttons above (e.g., Koyeb) and link your GitHub account.
  4. Configure Variables: Copy the necessary variables from the Environment Variables section below and paste them into your deployment dashboard.
  5. Deploy & Enjoy: Click 'Deploy', wait for the build process to complete, and your bot will come online!

Environment Variables

Required Configuration (.env)

Configure these variables in your deployment platform (Koyeb/Heroku/Railway) before starting the bot.

.env Variables
# Core Settings
SESSION_ID="your_session_id_here"
PREFIX="."
BOT_NAME="CIPTER-MD"
OWNER_NUMBER="94712345678"

# Features Toggle (true/false)
AUTO_READ_STATUS="true"
AUTO_TYPING="false"
AUTO_RECORDING="false"
ALWAYS_ONLINE="true"
REJECT_CALLS="true"

# Security & Group
ANTI_LINK="true"
ANTI_BAD_WORDS="true"
ANTI_BOT="false"
AUTO_WELCOME="true"

# APIs (Optional)
OPENAI_API_KEY=""
REMOVEBG_API_KEY=""

Manual Deployment

(Docker Method)

Prerequisites

  • Docker & Docker Compose installed on your system.

Setup & Run

bash
# Clone the repository
git clone https://github.com/sh4lu-z/cipter-md.git
cd cipter-md

# Create and configure .env file
cp .env.example .env

# Build and start the container
docker-compose up -d
